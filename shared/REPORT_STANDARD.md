# 日报公共标准

所有日报模块至少遵守以下规则。

## 内容质量

1. 先筛选重要性，再生成内容。
2. 不为了栏目完整而凑数。
3. 将事实、媒体观点和编辑判断明确区分。
4. 重要结论优先使用官方或一手来源。
5. 单一来源且无法交叉验证的信息必须标注。
6. 新闻必须解释背景和机制，避免只改写标题。
7. 与过去 7 天内容去重；持续事件只写新增变化。
8. 生成前读取模块的 `history-index.json`，先比较主题、实体、事件键、原句和来源 URL，再决定是否采用。

## 时间范围

- 默认覆盖最近 24 小时。
- 重大但更新频率较低的内容可回溯 7 天，并注明日期。
- 不把旧闻包装成当日新闻。

## 输出质量

- 使用清晰的 Markdown 层级。
- 标题直接表达内容，不使用夸张标题。
- 关键事实附近放置来源链接。
- 日期以模块配置的时区为准。
- 所有链接应直接指向原始页面，不指向搜索结果页。

## 归档规则

- 每日文件：archive/YYYY/MM/YYYY-MM-DD.md
- 最新一期：latest.md
- 每次成功生成后同时写入当日文件并更新 latest.md。
- 已存在的当日文件应更新，不重复创建第二份。
- 每期必须包含 `dedup` YAML 元数据，并为每个独立事件提供稳定、可读的 `content_keys`。
- 写入归档后运行 `npm run index:write`，再运行 `npm run index:check`；索引校验失败不得声称归档成功。
- 持续事件必须只写新增事实，并使用新的 `content_key`；必要时用 `allow_repeat: true` 与 `repeat_reason` 说明为什么需要再次讨论同一主题。
