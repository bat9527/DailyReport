# 金融科技出海日报

## 定位

面向个人能力成长和行业视野拓展的金融科技学习日报。

以非洲为核心，持续理解金融科技、数字银行、支付、信贷、风险管理和中国企业出海。它不是工作周报，不追踪个人项目进度，也不强行给出任务清单。

## 覆盖权重

- 非洲金融科技：约 60%
- 中国企业及金融科技能力出海：约 20%
- 全球标杆案例与前沿趋势：约 20%

权重是筛选方向，不要求每天机械凑齐。

## 时效原则

- 正常只收录生成时点前 24 小时的新信息，重要遗漏最多补充到 48 小时。
- 搜索不足时扩大国家、主题、公司和信源范围，不扩大时间窗口。
- 48 小时之外的材料只能作为新事件的简短背景，不能成为独立条目。
- 当天没有高价值新增时可以生成短报或明确写“暂无高价值新增”，不使用旧案例补数量。
- 从 2026-07-20 起，索引校验会阻止日期超限或缺少日期后缀的金融科技 `content_key`。

## 核心主题

1. 非洲金融监管与市场变化
2. 数字银行、移动钱包和支付网络
3. 数字信贷、普惠金融与商业模式
4. 风控、反欺诈、KYC、征信和催收
5. 银行核心、开放银行、云和 AI 应用
6. 中国公司、技术和模式的海外落地
7. 值得研究的公司、产品、失败案例和数据

## 阅读方式

- 对话推送：约 5 分钟读完，聚焦最重要的内容。
- GitHub 完整版：按当天真实信息量决定长度，包含必要背景、机制和完整来源。
- latest.md：始终指向或保存最新一期。
- archive：按照北京时间日期长期归档。

## 不包含

- 个人项目进度
- 日常工作任务提醒
- 没有实质信息的公司宣传稿
- 缺乏来源的市场传闻
- 为了数量而拼凑的普通新闻
- 超过 48 小时、被重新包装成当日进展的旧闻或旧案例
