# DailyReport 项目规划

## 1. 项目定位

DailyReport 是个人兴趣日报平台，不是一份固定的综合新闻。

平台解决三个问题：

1. 将持续关注的领域沉淀为独立日报。
2. 每天自动完成信息发现、核验、解释、推送和归档。
3. 兴趣变化时，可以低成本新增、暂停或调整日报。

## 2. 核心模型

平台 → 大分类 → 日报模块 → 每日归档

例如：

- 金融科技
  - 金融科技出海日报
- 语言学习
  - 英文日报
- 科技
  - 科技日报
- 人文
  - 历史日报

大分类用于导航；日报模块是真正独立运行的最小单元。

## 3. 模块边界

每个日报自行维护：

- 内容范围和排除项
- 重点地区和主题权重
- 信息来源
- 生成提示词
- 生成时间
- 输出模板
- latest.md
- archive 归档

根目录只维护注册表、导航和公共标准，不混放具体日报内容。

## 4. 输出方式

每次运行产生两层内容：

### 对话推送

- 约 5 分钟读完
- 先给最值得看的内容
- 保留必要来源链接
- 引导查看 GitHub 完整版

### GitHub 归档

- 约 15–20 分钟读完
- 补充背景、机制、案例、数据和来源
- 保存到对应日报模块的 archive 目录
- 同步更新 latest.md

## 5. 新增日报 SOP

1. 明确兴趣方向和希望解决的学习问题。
2. 在 reports 下建立独立模块。
3. 创建 README、config、PROMPT、sources、template 和 archive。
4. 在 reports/registry.yml 注册模块。
5. 创建独立自动化任务。
6. 试运行 7 天，根据阅读体验调整。
7. 稳定后保持运行；失去兴趣时暂停而不是删除历史。

## 6. 演进路线

### 第一阶段：框架验证

- 初始化平台
- 启用“金融科技出海”
- 验证每天生成、推送、归档链路

### 第二阶段：内容优化

- 根据实际阅读反馈调整篇幅和栏目
- 使用仓库内历史索引完成跨日去重，并持续跟踪重要事件
- 为高价值主题增加案例库

### 第三阶段：扩展兴趣

- 将验证有效的英文日报、科技日报迁入本仓库
- 按统一模块规范继续增加新分类
- 在根目录形成所有日报的统一导航

## 7. 成功标准

- 连续运行稳定
- 内容不凑数、不重复
- 每期都有可追溯来源
- 读完能增加知识、理解行业机制或拓展视野
- 新增一个日报不需要重构已有模块
